## Detailed Design Plan: REQ-16 / REQ-17 / REQ-18 / REQ-19 / REQ-20 / REQ-21 / REQ-22 / REQ-23 / REQ-24 / NFR-05 / NFR-06 (Backend Valuation Engine)

## Role

- Lead Backend Engineer / Python Architect

    - _Responsibility:_ Design and implement the FastAPI backend service — including the POST endpoint, Pydantic schema validation layer, deterministic valuation math engine, and modular file separation — ensuring all calculations are auditable, testable, and complete within 50ms.

## Task

- REQ-16 (Asynchronous Request Gateway): Expose a POST endpoint at `/api/v1/calculate` that accepts a JSON body matching the `ValuationRequest` schema, processes it through the engine, and returns a `ValuationResponse` JSON payload.
- REQ-17 (Pydantic Layer Rejection Handling): Use FastAPI's built-in Pydantic validation on the request body. When validation fails (e.g., wrong types, missing fields), return HTTP 422 with auto-generated error details — no custom validation code needed.
- REQ-18 (Base Valuation Branch Allocation): Map `startupStage` to a dollar baseline: Idea → $1,500,000, MVP/Prototype Built → $2,500,000, Early Traction → $4,000,000.
- REQ-19 (Pillar Modifier Logic - Team): Compute `s_team` starting at 1.0. Mixed background → 1.2. Prior exits → +0.3. Cap at 1.5.
- REQ-20 (Pillar Modifier Logic - Traction): Compute `s_traction` based on stage + revenue. Early Traction with revenue > 0 → 1.3. MVP → 1.0. Idea → 0.5. Retention > 80% → +0.15. Cap at 1.5.
- REQ-21 (Pillar Modifier Logic - Risk & Solo Penalties): Compute `s_risk` from runway. ≥12 months → 1.2. 4–11 months → 1.0. ≤3 months → 0.5. Solo founder (1 founder) → `min(s_risk, 0.5)`.
- REQ-22 (Mathematical Bounds Engine Guardrails): Clamp `v_final` to [$500,000, $6,500,000] after multiplier application.
- REQ-23 (Output Target Range Division): Produce Low = `v_final × 0.80`, Base = `v_final`, High = `v_final × 1.20`.
- REQ-24 (Deterministic Confidence Computation): Start at 70%. Add +5 for each non-zero field: `monthlyRevenueUSD`, `retentionRatePct`, `estimatedMarketSizeTAM`. Cap at 95%.
- NFR-05 (Backend Logic Performance): The synchronous math in `calculate_valuation()` must complete in under 50ms for any valid input.
- NFR-06 (Engineering Code Modularity): The backend must be split into exactly three files: `schemas.py` (Pydantic models), `engine.py` (pure math), `main.py` (FastAPI routes). No circular imports.

## Context

- The backend is the brain of PreSeedIQ. It receives structured form data from the frontend, runs a deterministic Berkus/Scorecard hybrid formula, and returns a valuation range plus qualitative analysis.
- The formula is entirely deterministic — no randomness, no external API dependency for the core math. Given the same inputs, it always produces the same outputs. This makes it fully unit-testable without mocks.
- Pydantic validation (REQ-17) means the endpoint automatically rejects malformed payloads at the framework level. The engine never sees invalid data.
- The modular split (NFR-06) means `engine.py` can be unit-tested by constructing `ValuationRequest` objects directly — no HTTP context needed. Routes in `main.py` are thin wrappers.
- Performance (NFR-05) is trivially met: the engine performs ~20 arithmetic ops, a few string comparisons, and no I/O. Even in CPython, this completes in under 1ms.

## Constraints

- Framework: FastAPI running under `uvicorn`. No Django, Flask, or alternative frameworks.
- Language: Python 3.11+ (as defined in `environment.yml`).
- Validation: All request validation must use Pydantic field constraints (`Field(..., ge=0)`, `Literal[...]`, `min_length`, etc.). No hand-written `if` checks in the route handler.
- Zero Database: As per NFR-01, the backend must not import or reference any database driver, ORM, cache client, or file I/O. The entire data lifecycle is request-scoped.
- Formula Finality: The weighted multiplier formula is fixed per the PRD: `M = (S_team × 0.30) + (S_traction × 0.30) + (S_market × 0.20) + (S_risk × 0.20)`. The `s_financial` score is computed for the dashboard bars but does NOT appear in the multiplier formula.
- Return Shape: The response dict from `calculate_valuation()` must have exactly three top-level keys: `valuation`, `subPillars`, `profileAnalysis`. The route handler wraps this into a `ValuationResponse` Pydantic model before returning.
- Modular Boundaries: `engine.py` imports `schemas.py` only. `main.py` imports both `schemas.py` and `engine.py`. Neither imports from the other's layer (`main.py` does not import `engine` functions that call `main`).

## Format

The implementation spans three files (already exist, verify only):

1. **`backend/src/schemas.py`** — Pydantic models: 5 step models + `ValuationRequest` + `ValuationRange` + `SubPillars` + `ProfileAnalysis` + `ValuationResponse`.
2. **`backend/src/engine.py`** — Pure function `calculate_valuation(payload: ValuationRequest) -> dict`. No I/O, no imports beyond `schemas`.
3. **`backend/src/main.py`** — FastAPI app with CORS middleware, single POST route at `/api/v1/calculate`, Ollama AI circuit breaker, error handling.
4. **`frontend/src/services/api.js`** — Client-side `fetch` wrapper that POSTs to the backend.

## Technical Configuration Maps (Data Structures)

### A. Pydantic Schema Layer (`schemas.py`)

```
Step1BasicInfo
├── companyName: str (min_length=1)
├── countryCode: str (min_length=2, max_length=3)
├── industry: str
├── businessModel: Literal["B2B", "B2C"]
└── startupStage: Literal["Idea", "MVP/Prototype Built", "Early Traction"]

Step2TractionPerformance
├── monthlyRevenueUSD: float (ge=0)
├── revenueGrowthRatePct: float (ge=0)
├── numberOfUsersOrCustomers: int (ge=0)
├── growthType: str
├── growthRatePct: float (ge=0)
└── retentionRatePct: float (ge=0, le=100)

Step3MarketIndustry
├── estimatedMarketSizeTAM: float (ge=0)
├── industryGrowthRate: Literal["Low", "Moderate", "High"]
└── competitiveIntensity: Literal["Low", "Medium", "High"]

Step4Team
├── numberOfFounders: int (ge=1)
├── founderBackground: Literal["Technical Only", "Business Only", "Mixed"]
└── priorExitsOrRelevantExperience: bool

Step5FinancialRisk
├── burnRateUSDPerMonth: float (ge=0)
├── runwayMonths: int (ge=0)
├── monetizationClarity: Literal["Clear/Validated", "Hypothetical", "Unclear"]
└── regulatoryOrExecutionRisk: Literal["Low", "Medium", "High"]

ValuationRequest
├── step1BasicInfo: Step1BasicInfo
├── step2TractionPerformance: Step2TractionPerformance
├── step3MarketIndustry: Step3MarketIndustry
├── step4Team: Step4Team
└── step5FinancialRisk: Step5FinancialRisk
```

### B. Engine Math Flow (`engine.py`)

```
calculate_valuation(ValuationRequest)
│
├── 1. BASE ASSIGNMENT (REQ-18)
│     stage = payload.step1BasicInfo.startupStage
│     Idea             → v_base = $1,500,000
│     MVP/Prototype    → v_base = $2,500,000
│     Early Traction   → v_base = $4,000,000
│
├── 2. PILLAR SCORES (REQ-19, 20, 21)
│
│   s_team (weight 30%):
│     baseline = 1.0
│     if background == "Mixed" → 1.2
│     if priorExits → +0.3
│     cap at 1.5
│
│   s_traction (weight 30%):
│     if stage == "Early Traction" AND revenue > 0 → 1.3
│     elif stage == "MVP/Prototype Built" → 1.0
│     else → 0.5
│     if retention > 80% → +0.15
│     cap at 1.5
│
│   s_market (weight 20%):
│     Low competition → 1.2
│     Medium → 1.0
│     High → 0.6
│
│   s_risk (weight 20%):
│     runway ≥ 12 → 1.2
│     runway 4–11 → 1.0
│     runway ≤ 3 → 0.5
│     if founders == 1 → min(s_risk, 0.5)
│
│   s_financial (not in multiplier — for dashboard only):
│     Clear/Validated → 1.2
│     Hypothetical → 1.0
│     Unclear → 0.5
│
├── 3. SYNTHESIS (REQ-22)
│     M = (s_team × 0.30) + (s_traction × 0.30) + (s_market × 0.20) + (s_risk × 0.20)
│     v_final = v_base × M
│     clamp(v_final, 500000, 6500000)
│
├── 4. RANGE DIVISION (REQ-23)
│     low  = round(v_final × 0.80, 2)
│     base = round(v_final, 2)
│     high = round(v_final × 1.20, 2)
│
├── 5. CONFIDENCE (REQ-24)
│     confidence = 70
│     if monthlyRevenueUSD > 0     → +5
│     if retentionRatePct > 0      → +5
│     if estimatedMarketSizeTAM > 0 → +5
│     cap at 95
│
├── 6. PROFILE ANALYSIS
│     Build strengths/weaknesses arrays from score thresholds
│
└── return { valuation, subPillars, profileAnalysis }
```

### C. API Route (`main.py`)

```
POST /api/v1/calculate
  Headers: Content-Type: application/json
  Body: ValuationRequest (validated by Pydantic automatically → 422 on failure)
  
  1. result = calculate_valuation(payload)          # <1ms, pure math
  2. try: Ollama AI prompt (5.0s timeout)           # optional, circuit-breakered
  3. return ValuationResponse(
       valuation=result["valuation"],
       subPillars=result["subPillars"],
       profileAnalysis=result["profileAnalysis"],
       aiRecommendation=...                          # Ollama response or fallback text
     )
  
  On unhandled exception: HTTP 500 with detail string
```

### D. Client API Module (`api.js`)

```javascript
export async function submitValuationData(formData) {
  const response = await fetch(`${BACKEND_URL}/api/v1/calculate`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(formData),
  });
  if (!response.ok) throw new Error(...);
  return response.json();
}
```

### E. Modular File Dependency Graph (NFR-06)

```
api.js (frontend) ───HTTP POST──→ main.py
                                      │
                            imports ──┤
                             ┌───────┴────────┐
                             ▼                 ▼
                         schemas.py        engine.py
                         (Pydantic)        (pure math)
                                              │
                                        imports ┤
                                              ▼
                                         schemas.py
```

`schemas.py` is a leaf node (no imports beyond `pydantic`). `engine.py` imports only `schemas.py`. `main.py` imports `schemas.py` and `engine.py`. No circular dependencies.

## Acceptance Criteria

- AC-01 (API Gateway — maps to REQ-16): POSTing a valid `ValuationRequest` JSON body to `/api/v1/calculate` must return HTTP 200 with a JSON body matching `ValuationResponse` shape.
  - _Verification:_ Use `httpx.AsyncClient` with `ASGITransport` to POST a valid payload. Assert `response.status_code == 200` and `response.json()` contains keys `valuation`, `subPillars`, `profileAnalysis`, `aiRecommendation`.
  - _Test Coverage:_ `test_asynchronous_request_gateway` (REQ-16, NFR-05).

- AC-02 (Pydantic Rejection — maps to REQ-17): POSTing a malformed payload (e.g., empty `step1BasicInfo` or negative `monthlyRevenueUSD`) must return HTTP 422 with auto-generated validation error details.
  - _Verification:_ POST `{"step1BasicInfo": {}}`. Assert `response.status_code == 422`. POST `{"step1BasicInfo": {...valid..., "monthlyRevenueUSD": -1}}`. Assert 422.
  - _Test Coverage:_ `test_pydantic_layer_rejection_handling` (REQ-17).

- AC-03 (Base Valuation — maps to REQ-18): Passing `startupStage` = "Idea" must produce `v_base` >= $1,500,000. "MVP/Prototype Built" >= $2,500,000. "Early Traction" >= $4,000,000.
  - _Verification:_ Call `calculate_valuation` with each stage. Assert `result["valuation"]["base"]` meets the respective minimum.
  - _Test Coverage:_ `test_base_valuation_branch_allocation` (REQ-18).

- AC-04 (Team Modifier — maps to REQ-19): A payload with `Mixed` background and `priorExitsOrRelevantExperience=True` must produce `subPillars.teamScore == 1.5`.
  - _Verification:_ Construct payload with those values. Assert `result["subPillars"]["teamScore"]` is capped at 1.5. Assert profile analysis contains a strength entry.
  - _Test Coverage:_ `test_pillar_modifier_team_logic` (REQ-19).

- AC-05 (Traction Modifier — maps to REQ-20): A payload with `retentionRatePct=85.0` must trigger the +0.15 bonus. Assert the final `tractionScore` reflects the increment.
  - _Verification:_ Compare two payloads identical except for retention rate (one at 80, one at 85). Assert the 85-payload's `tractionScore` is exactly 0.15 higher. Assert profile analysis includes the retention strength string.
  - _Test Coverage:_ `test_pillar_modifier_traction_retention_bonus` (REQ-20).

- AC-06 (Risk & Solo Penalty — maps to REQ-21): A payload with `runwayMonths=2` must cap `riskScore` at 0.5. A payload with `numberOfFounders=1` must also cap `riskScore` at 0.5 regardless of runway length.
  - _Verification:_ Set `runwayMonths=2`, assert `riskScore == 0.5`. Set `numberOfFounders=1` with `runwayMonths=24`, assert `riskScore == 0.5`. Assert weaknesses array includes solo-founder warning.
  - _Test Coverage:_ `test_pillar_modifier_risk_solo_penalties` (REQ-21).

- AC-07 (Bounds Guardrails — maps to REQ-22): A payload with worst-case parameters must not produce a valuation below $500,000. A payload with best-case parameters must not exceed $6,500,000.
  - _Verification:_ Construct worst-case payload (Idea, all minimums), assert `base >= 500000`. Construct best-case (Early Traction, max revenue, max retention, all bonuses), assert `base <= 6500000`.
  - _Test Coverage:_ `test_mathematical_bounds_engine_guardrails` (REQ-22).

- AC-08 (Range Division — maps to REQ-23): The returned `low` must equal `round(base × 0.80, 2)`. The returned `high` must equal `round(base × 1.20, 2)`.
  - _Verification:_ Call engine, assert `result["valuation"]["low"] == round(result["valuation"]["base"] * 0.80, 2)` and `result["valuation"]["high"] == round(result["valuation"]["base"] * 1.20, 2)`.
  - _Test Coverage:_ `test_output_target_range_division` (REQ-23).

- AC-09 (Confidence — maps to REQ-24): An empty data profile must return `confidencePct == 70`. Populating all three tracked fields sequentially must raise it to 85.
  - _Verification:_ Call engine with zeros (except required fields), assert `confidencePct == 70`. Set `monthlyRevenueUSD=1000`, assert `75`. Set `retentionRatePct=50`, assert `80`. Set `estimatedMarketSizeTAM=1000000`, assert `85`.
  - _Test Coverage:_ `test_deterministic_confidence_computation` (REQ-24).

- AC-10 (Performance — maps to NFR-05): The `calculate_valuation()` function must complete in under 50ms for any valid input.
  - _Verification:_ Time the function call across 100 iterations with varied inputs. Assert mean execution time < 50ms. (In practice, CPython executes this in <1ms — the threshold is trivially met.)
  - _Test Coverage:_ `test_asynchronous_request_gateway` (NFR-05 — end-to-end latency tracking).

- AC-11 (Modularity — maps to NFR-06): The backend must have exactly three `.py` files in `backend/src/` — `schemas.py`, `engine.py`, `main.py`. `engine.py` imports only from `schemas`. `main.py` imports from both `schemas` and `engine`. No file imports from the other direction.
  - _Verification:_ Audit import statements in all three files. Assert no circular imports exist. Assert `engine.py` contains no FastAPI references. Assert `main.py` route handler delegates math to `engine.calculate_valuation()`.
  - _Test Coverage:_ Architecture file structure review (no dedicated unit test).
