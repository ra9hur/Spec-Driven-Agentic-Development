## Detailed Design Plan: REQ-07 / REQ-08 / REQ-09 / REQ-10 / REQ-11 / REQ-12 / REQ-13 / REQ-14 / REQ-15 / NFR-03 / NFR-04 (5-Step Form Wizard)

## Role

- Lead Frontend Engineer / UI Architect

    - _Responsibility:_ Build the 5-step progressive form wizard at the core of the data collection experience — including step routing, field-level validation per step, numeric input filtering, responsive mobile layout, "CALCULATE VALUATION" button swap, and session-backed state persistence across navigation.

## Task

- REQ-07 (Wizard Layout Flow): Create a 5-step progressive form panel backed by a visual step-progress indicator (numbered circles 1–5). Each step is a distinct route (`/step/1` through `/step/5`) with "Continue" advancing to the next step and "Back" returning to the previous.
- REQ-08 (Responsive Reflow Constraints): Ensure all horizontally arrayed form columns and dashboard grids collapse to single-column vertical stacks on viewports narrower than 768px.
- REQ-09 (Step 1 Validation): 5 fields — Company Name, Country (dropdown), Industry (dropdown), Business Model (B2B/B2C dropdown), Startup Stage (Idea/MVP/Early Traction dropdown). "Continue" disabled until all 5 filled.
- REQ-10 (Step 2 Validation): 6 fields — Monthly Revenue (`$` prefix), Revenue Growth Rate (%), Number of Users, Growth Type (MoM/YoY dropdown), Growth Rate (%), Retention Rate (%). "Continue" disabled until all 6 filled.
- REQ-11 (Step 3 Validation): 3 fields — TAM, Industry Growth Rate (Low/Moderate/High dropdown), Competitive Intensity (Low/Medium/High dropdown). "Continue" disabled until all 3 filled.
- REQ-12 (Step 4 Validation): 3 fields — Number of Founders (integer spinner), Founder Background (Technical Only/Business Only/Mixed dropdown), Prior Exits (checkbox). "Continue" disabled until spinner + dropdown filled.
- REQ-13 (Step 5 Validation): 4 fields — Burn Rate (`$`), Runway (Months), Monetization Clarity (Clear/Validated/Hypothetical/Unclear dropdown), Regulatory Risk (Low/Medium/High dropdown). "CALCULATE VALUATION" button disabled until all 4 filled.
- REQ-14 (String Input Filtering): Numeric input fields must strip non-numeric characters (letters, special chars, negative signs) from keystroke input dynamically using a regex filter.
- REQ-15 (Submission Trigger Action UI): On step 5, the primary action button label changes from "Continue" to "CALCULATE VALUATION".
- NFR-03 (Zero-Tutorial Usability): Layout must use standard form patterns (label above input, clear section headers, consistent spacing) so a non-technical user can complete all steps without guidance.
- NFR-04 (Navigation State Persistence): Navigating "Back" from a later step must restore all previously entered values. The form context persists across all step transitions within the same session.

## Context

- The 5-step wizard is the primary data collection surface of PreSeedIQ. Every field maps to a specific parameter in the backend `ValuationRequest` Pydantic schema.
- Breaking the form into 5 steps reduces cognitive load versus a single long scrolling page — critical for non-technical founders using the tool on mobile devices.
- Step-level validation (locking "Continue" until all fields on that step are populated) prevents incomplete submissions and guides users through a linear flow.
- State persistence (NFR-04) is essential for the UX pattern of allowing users to go "Back" to adjust a prior answer without losing data entered on later steps.
- REQ-14 is a defense-in-depth measure: even though the backend Pydantic schema rejects type mismatches, filtering at the keystroke level provides instant visual feedback and prevents confusion.

## Constraints

- Step Count: Exactly 5 steps. No more, no fewer. The progress indicator must display circles numbered 1–5.
- Routing: Each step must be a distinct route (`/step/:stepId`) with `FormWizard` as the container component that uses `useParams` to determine the current step. Invalid step IDs must redirect to `/step/1`.
- Progress Indicator Styling: Current step → `bg-brand-accent` (`#E50914`) filled circle. Completed steps → `bg-muted-highlight` (`#9B050C`). Future steps → `bg-secondary-surface` (`#161616`) with `border-border-default` (`#262626`) outline and muted text.
- Validation Pattern: Each step component defines a `FIELDS` array and computes `allFilled` via `useMemo` — every field must be non-empty (string length > 0) before "Continue" enables. Step 4 skips the checkbox from `allFilled` (checkbox is optional).
- Button Labels: Steps 1–4 show "Continue". Step 5 shows "CALCULATE VALUATION". Steps 2–5 show "Back". Step 1 has no "Back".
- Responsive Breakpoint: The Tailwind `md` breakpoint (768px) must be used for responsive grids. Any `grid-cols-*` layout on screens ≥768px must collapse to `grid-cols-1` on screens <768px.
- Numeric Filter Scope: Fields with `inputMode="decimal"` in `FormInput.jsx` must apply a regex filter that strips everything except digits, decimal points, and leading minus signs. The spinner (`type="number"`) is exempt (native browser behavior handles it).
- NFR-03 Convention: All step components must follow the same structural pattern: `<h2>` header → `<div className="flex flex-col gap-4">` with `<FormInput>` children → action buttons row with `justify-between`.

### C. State Data Flow

```
FormWizard mount
       │
       ▼
useState(() => loadFormData() || {})
       │
       ▼
formData = { step1BasicInfo: {...}, step2TractionPerformance: {...}, ... }
       │
       ▼  Passed as `data` prop to active StepComponent
StepComponent reads data[KEY] || {}
       │
       ▼  User changes a field
updateField(KEY, field, value)
       │
       ▼  setFormData(prev => ({...prev, [KEY]: {...prev[KEY], [field]: value}}))
useEffect → saveFormData(formData)  [NFR-04]
       │
       ▼  User clicks Continue/Back
navigate(`/step/${nextStep}`)
       │
       ▼  FormWizard re-mounts at new step
useState(() => loadFormData() || {})  →  all previous data restored
```

### D. Lift-and-Shift: Step Pattern Template

Every step component follows this exact contract:

```javascript
const KEY = 'stepXSomeName';
const FIELDS = [...];

export default function StepX({ data, updateField, onNext, onBack }) {
  const values = data[KEY] || {};
  const allFilled = useMemo(() => FIELDS.every(f => {
    const v = values[f];
    return v !== undefined && v !== null && v !== '';
  }), [values]);

  return (
    <div>
      <h2 className="text-xl font-semibold text-text-primary mb-6">Step Title</h2>
      <div className="flex flex-col gap-4">
        {...fields}
      </div>
      <div className="flex justify-between mt-6">
        {onBack && <Button variant="secondary" onClick={onBack}>Back</Button>}
        <Button disabled={!allFilled} onClick={onNext}>Continue</Button>
      </div>
    </div>
  );
}
```
