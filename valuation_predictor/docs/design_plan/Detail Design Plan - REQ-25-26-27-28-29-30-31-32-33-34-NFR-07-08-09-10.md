## 1. REQ-25 (Ollama Local AI Prompt Delivery)

**Specification:** Backend structures a prompt payload outlining user inputs and asynchronously transmits it to the local Ollama endpoint at `http://localhost:11434/v1/chat/completions`.

**Implementation (already in `backend/src/main.py:34-51`):**
- Builds a `prompt_payload` dict with `model` (from `OLLAMA_MODEL` env var, default `llama3.2`) and `messages` containing a user-role message.
- The prompt includes company name, startup stage, LOW/BASE/HIGH valuation, and strengths/weaknesses.
- Uses `httpx.AsyncClient` to POST to `{OLLAMA_BASE_URL}/v1/chat/completions`.
- Response parsed as `res_json["choices"][0]["message"]["content"]`.

**Edge Cases:**
- Ollama not installed or not running → captured by REQ-27 fallback.
- Model name invalid → Ollama returns error, captured by circuit breaker.
- Empty response from Ollama → try/except catches `KeyError`.

**Verification:**
- `test_ollama_api_transmission` in backend tests.

## 2. REQ-26 (AI Fault-Tolerant Circuit Breaker)

**Specification:** Backend transaction executes inside a `try/except` wrapper with a 5.0-second timeout.

**Implementation (already in `backend/src/main.py:53-64`):**
- Inner `try/except` wraps the `httpx.AsyncClient.post` call.
- `timeout=5.0` on the client POST.
- Catches `httpx.TimeoutException`, `httpx.RequestError`, and `KeyError`.
- On exception, silently falls through without crashing — `ai_recommendation` retains its initial fallback value.

**Edge Cases:**
- Network timeout exactly at 5.0s boundary → httpx raises `TimeoutException`.
- Partial response received but connection drops → `RequestError`.
- Ollama returns malformed JSON → `KeyError` on `res_json["choices"]`.

**Verification:**
- `test_ai_fault_tolerant_circuit_breaker` (simulated latency).

## 3. REQ-27 (Network Timeout Fallback Execution)

**Specification:** If Ollama fails or exceeds 5.0s, engine uses hardcoded fallback text without crashing.

**Implementation (already in `backend/src/main.py:28-32`):**
- `ai_recommendation` initialized to a 4-part fallback text block before the try block.
- If circuit breaker activates, `ai_recommendation` is never overwritten → fallback is returned.
- Fallback covers: Fundraising Readiness, Valuation Expectations, Recommended Actions, Critical Concerns/Opportunities.

**Edge Cases:**
- All Ollama failure modes (offline, timeout, bad response, non-200 status).
- Empty response from Ollama with 200 status → `KeyError` triggers fallback.

**Verification:**
- `test_network_timeout_fallback_execution`.

## 8. REQ-32 (Action Bar Functional Handlers)

**Specification:** Fixed bottom bar with 4 buttons: BACK TO FORM, EXPORT, SHARE, NEW VALUATION.

**Implementation (already in `Dashboard.jsx:137-145`):**
- `fixed bottom-0 left-0 right-0 bg-secondary-surface border-t border-border-default`.
- Four buttons:
  - **BACK TO FORM** (`onClick → navigate('/step/5')`) — uses React Router navigation, form data persists via sessionStorage.
  - **EXPORT** (`onClick → window.print()`) — triggers browser print dialog, NFR-10 print CSS takes effect.
  - **SHARE** (`onClick → navigator.clipboard.writeText(url)`) — copies results page URL to clipboard.
  - **NEW VALUATION** (`onClick → purgeUserSession()`) — calls `sessionStorage.clear()` + `window.location.href = '/step/1'`.

**Edge Cases:**
- SHARE: clipboard API unavailable → caught by try/catch, silently fails.
- EXPORT: print stylesheet reverses bg to white.
- NEW VALUATION: hard redirect (not React Router navigate) ensures full state wipe.

**Verification:**
- `test_action_bar_functional_handlers`.

## 12. NFR-08 (Fallback Turnaround Budget)

**Specification:** Total roundtrip <120ms when running on local fallback.

**Implementation (already in `backend/src/main.py`):**
- When Ollama is unavailable, `httpx.TimeoutException` or `RequestError` fires within 5.0s.
- The try/except is instant — no retry logic.
- Synchronous engine calculation (`calculate_valuation`) completes in <50ms (NFR-05).
- Total response: engine (~1ms) + failed Ollama (5.0s timeout) — but the circuit breaker exits the httpx call immediately on connection refusal (OS error), which is typically <100ms.
- Fallback path skips Ollama entirely.

**Verification:**
- `test_network_timeout_fallback_execution`.
